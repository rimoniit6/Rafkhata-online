## ১. প্রোডাক্ট ওভারভিউ

### ১.১ প্রোডাক্ট নাম
**শিক্ষা বাংলা (Shiksha Bangla)**

### ১.২ উদ্দেশ্য
বাংলাদেশের ৬ষ্ঠ শ্রেণি থেকে HSC পর্যন্ত শিক্ষার্থীদের জন্য একটি সম্পূর্ণ বাংলা ভাষায় অনলাইন শিক্ষা প্ল্যাটফর্ম, যেখানে লেকচার, MCQ অনুশীলন, সৃজনশীল প্রশ্ন (CQ), বোর্ড পরীক্ষার প্রশ্ন, সাজেশন, পরীক্ষা এবং প্রতি-কন্টেন্ট পেমেন্ট সিস্টেম রয়েছে।

### ১.৩ টার্গেট ইউজার
| ভূমিকা | বর্ণনা |
|---------|--------|
| **শিক্ষার্থী (Student)** | ৬ষ্ঠ-HSC শ্রেণির শিক্ষার্থী, বিনামূল্যে ও প্রিমিয়াম কন্টেন্ট ব্যবহার করে |
| **অ্যাডমিন (Admin)** | কন্টেন্ট ম্যানেজমেন্ট, পেমেন্ট রিভিউ, ইউজার ম্যানেজমেন্ট |
| **সুপার অ্যাডমিন (Super Admin)** | সম্পূর্ণ সিস্টেম নিয়ন্ত্রণ, ডাটাবেস ম্যানেজমেন্ট |

### ১.৪ পেমেন্ট মডেল
**প্রতি-কন্টেন্ট মূল্য (Per-Content Pricing)** — সাবস্ক্রিপশন প্ল্যান সরিয়ে দেওয়া হয়েছে। প্রতিটি প্রিমিয়াম কন্টেন্ট (লেকচার, MCQ, CQ, পরীক্ষা, সাজেশন, বান্ডেল, প্যাকেজ) আলাদাভাবে কেনা যায়।

**পেমেন্ট মেথড**: bKash, Nagad, Rocket (ম্যানুয়াল ট্রানজেকশন ভেরিফিকেশন)

## ৩. আর্কিটেকচার

### ৩.১ ওভারভিউ

```

│  └─────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

### ৩.২ রাউটিং আর্কিটেকচার

**গুরুত্বপূর্ণ**: এই প্রজেক্ট Next.js ফাইল-বেসড রাউটিং ব্যবহার করে না। পুরো অ্যাপটি একটি **Single Page Application (SPA)** যা `src/app/page.tsx` থেকে রেন্ডার হয়।

- **ক্লায়েন্ট-সাইড রাউটার**: `useRouterStore` (Zustand) — ৩৫+ রাউট পাথ
- **ডাইনামিক ইম্পোর্ট**: `next/dynamic` দিয়ে কোড স্প্লিটিং
- **হিস্ট্রি ম্যানেজমেন্ট**: `navigate()` ও `goBack()` মেথড
- **URL সিঙ্ক নেই**: ব্রাউজার URL পরিবর্তন হয় না, শুধু Zustand state পরিবর্তন হয়

### ৩.৩ অথেনটিকেশন ফ্লো

```
লগইন → POST /api/auth/login → JWT তৈরি → HttpOnly Cookie সেট → Zustand Auth Store আপডেট
                                                              ↓
                                                    পরবর্তী API কলে Cookie থেকে JWT ভেরিফাই
```

- **JWT Storage**: HttpOnly Cookie (`auth-token`) + Authorization Header fallback
- **JWT Payload**: `{id, email, role, isPremium}`
- **JWT Expiry**: 7 দিন
- **Middleware**: শুধুমাত্র `/api/*` রাউটে কাজ করে

### ৩.৪ গেটওয়ে (Caddyfile)

```
পোর্ট 81 → Next.js (পোর্ট 3000)
পোর্ট 81 → ?XTransformPort=XXXX → localhost:XXXX (মিনি সার্ভিস)
```

## ১০. মিডলওয়্যার কনফিগারেশন

### ১০.১ রাউট ক্লাসিফিকেশন

| ক্যাটাগরি | রাউট | অথ রিকোয়ারমেন্ট |
|----------|------|------------------|
| **Auth Routes** | `/api/auth/login`, `/api/auth/register`, `/api/auth/forgot-password`, `/api/auth/reset-password`, `/api/auth/logout` | সম্পূর্ণ পাবলিক (সকল মেথড) |
| **Content Routes** | classes, subjects, chapters, lectures, mcq, cq, bundles, suggestions, packages, board-questions, boards, years, courses, faqs, testimonials, banners, notices, exams, plans, search, payment/content-info, payment/check | GET পাবলিক, POST/PUT/DELETE/PATCH অথ প্রয়োজন |
| **Admin Routes** | `/api/admin/*` | admin বা super_admin রোল |
| **Super Admin Routes** | `/api/admin/database/*`, `/api/admin/ensure-super-admin` | শুধু super_admin রোল |

### ১০.২ JWT ফ্লো

1. Cookie (`auth-token`) বা Authorization Header থেকে টোকেন পড়া
2. `jwt.verify()` দিয়ে ভ্যালিডেশন
3. ইউজার ইনফো `x-user-id` ও `x-user-role` হেডারে ইনজেক্ট
4. ডাউনস্ট্রিম API রাউটে এই হেডার থেকে ইউজার তথ্য পড়া

### ১০.৩ প্রক্সি/মিডলওয়্যার আর্কিটেকচার (`src/proxy.ts`)

```
┌─────────────────────────────────────────────────────────────┐
│                    src/proxy.ts (Middleware)                 │
├─────────────────────────────────────────────────────────────┤
│  ১. Static/Next.js internals → Pass through                 │
│  ২. Public pages (/, /login, /register, /privacy, /terms)  │
│     → Pass through + Security Headers                       │
│  ৩. Public API routes → Pass through + Security Headers     │
│  ৪. Protected API routes → updateSession() → Verify JWT    │
│     ├─ No session → 401 UNAUTHORIZED                        │
│     ├─ Admin route + non-admin → 403 FORBIDDEN              │
│     └─ Super-admin route + non-super → 403 FORBIDDEN        │
│  ৫. Protected pages (non-API) → updateSession()            │
│     ├─ No session → Redirect /login?redirect=...           │
│     └─ Admin page + non-admin → Redirect /                  │
│  ৬. All responses → Security Headers (CSP, X-Frame, etc.)  │
└─────────────────────────────────────────────────────────────┘
```

**Security Headers যোগ করা হয়েছে:**
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Content-Security-Policy` (স্ট্রিক্ট পলিসি Supabase, UploadThing, CDN এর জন্য)

- **এন্ডপয়েন্ট**: `GET /api/csrf-token` — নতুন টোকেন জেনারেট করে কুকিতে সেট করে

## ২৬. ডেটা ফ্লো ডায়াগ্রাম

### ২৬.১ কন্টেন্ট ব্রাউজিং ফ্লো

```
হোমপেজ → শ্রেণি নির্বাচন → বিষয় তালিকা → অধ্যায় তালিকা
                                                  ↓
                                    ┌─────────────┼──────────────┐
                                    ↓             ↓              ↓
                              লেকচার লিস্ট    MCQ প্র্যাকটিস    CQ লিস্ট
                                    ↓             ↓              ↓
                              লেকচার ভিউ     MCQ পরীক্ষা     CQ ভিউ
                              (প্রিমিয়াম?)   (টাইমার)      (প্রিমিয়াম?)
                                    ↓             ↓
                              পেওয়াল/পেমেন্ট  রেজাল্ট
```

### ২৬.২ পেমেন্ট ফ্লো

```
প্রিমিয়াম কন্টেন্ট দেখুন
        ↓
পেওয়াল দেখান ("কিনুন" বাটন)
        ↓
PaymentPage → content-info API → কন্টেন্ট তথ্য (নাম, মূল্য)
        ↓
bKash/Nagad/Rocket-এ টাকা পাঠান
        ↓
Transaction ID + ফোন নম্বর জমা দিন
        ↓
POST /api/payment → পেমেন্ট রেকর্ড (pending)
        ↓
অ্যাডমিন → /admin/payments → Approve/Reject
        ↓
Approve → নোটিফিকেশন → কন্টেন্ট আনলক
Reject → নোটিফিকেশন → পেমেন্ট বাতিল
```

## ২৭. পারফরম্যান্স বিবেচনা

| সমস্যা | বর্তমান | সুপারিশ |
|--------|---------|---------|
| ডাটাবেস | SQLite (সিঙ্গেল-ফাইল) | বড় স্কেলে PostgreSQL-এ মাইগ্রেট |
| ক্যাশিং | ইন-মেমোরি (নোটিশ, সাজেশন) | Redis-এ মাইগ্রেট |
| রেট লিমিটিং | ইন-মেমোরি | Redis/Upstash-এ মাইগ্রেট |
| ফাইল স্টোরেজ | লোকাল `public/uploads/` | S3/Cloudflare R2-তে মাইগ্রেট |
| কোড স্প্লিটিং | `next/dynamic` ✅ | ইতিমধ্যে ভালো |
| পেজিনেশন | কিছু API-তে আছে | সব API-তে যোগ করুন |
| N+1 কুয়েরি | পেমেন্ট চেকে আছে | Batch API ব্যবহার করুন |
